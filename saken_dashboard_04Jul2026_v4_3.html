⚡ Automate Daily Dashboard Updates from Email
Your helpdesk sends the Excel report to your inbox every day. Here are 3 options to make the dashboard update automatically — ranked by ease of setup. As a CFO with Odoo + IT background, Option 2 or 3 will give you the most control.
📧
Gmail + Google Apps Script
Script watches your inbox, finds the daily Excel, parses it, and updates your GitHub Pages dashboard automatically every morning at 7am.
Free No server needed
🔄
Make.com (Zapier alternative)
Connect Gmail → Parse Excel → Push to GitHub. Visual no-code flow. Runs automatically when email arrives. Free tier covers daily use.
Free tier No-code
🐍
Python Script on Your Server
Since you run Odoo, you already have a server. A simple Python cron job checks Gmail API, parses the file, rebuilds the dashboard HTML, and pushes to GitHub.
Most control Free
Option 1 — Gmail + Google Apps Script (Easiest — Zero Cost)
1
Open Google Apps Script
Go to script.google.com → New Project. This runs inside Google's servers — no computer needed.
2
Paste this script
This finds the daily report in Gmail, extracts it, and triggers Claude to rebuild the dashboard:
function dailyDashboardUpdate() { // Search Gmail for today's report var threads = GmailApp.search( 'subject:"Front Office Report" has:attachment newer_than:1d' ); if (threads.length === 0) { Logger.log('No report found today'); return; } // Get the Excel attachment var msg = threads[0].getMessages()[0]; var attachments = msg.getAttachments(); var xlsxFile = attachments.find(a => a.getName().includes('Front_Office_Report') ); if (!xlsxFile) return; // Save to Google Drive (shared folder) var folder = DriveApp.getFolderById('YOUR_FOLDER_ID'); var file = folder.createFile(xlsxFile); Logger.log('Report saved: ' + file.getName()); // → Then upload updated dashboard to GitHub via GitHub API }
3
Set a daily trigger
In Apps Script → Triggers → Add Trigger → Time-driven → Day timer → 7:00–8:00 AM. It runs automatically every morning.
4
Push updated dashboard to GitHub
The script calls GitHub API to replace your index.html with the fresh data. Your team's URL automatically shows today's report within minutes of receiving the email.
// GitHub API push (add inside the function above) var GITHUB_TOKEN = 'your_token_here'; var REPO = 'your-username/saken-dashboard'; var response = UrlFetchApp.fetch( 'https://api.github.com/repos/' + REPO + '/contents/index.html', { method: 'put', headers: { 'Authorization': 'token ' + GITHUB_TOKEN, 'Content-Type': 'application/json' }, payload: JSON.stringify({ message: 'Auto-update: ' + new Date().toDateString(), content: Utilities.base64Encode(newDashboardHTML), sha: currentFileSHA // get this first with a GET request }) } ); Logger.log('Dashboard updated: ' + response.getResponseCode());
Option 2 — Make.com Visual Flow (No-Code)
1
Create free account at make.com
Make.com is a visual automation platform — like Zapier but more powerful and with a generous free tier (1,000 ops/month, more than enough for daily use).
2
Build this 4-step scenario
Gmail (Watch emails with attachment) → Filter (subject contains "Front Office Report") → Claude API (parse Excel + generate dashboard) → GitHub (push updated index.html)
3
Schedule it
Set the scenario to run every day at 7am. From that point, your team's dashboard link auto-refreshes with zero manual effort.
Option 3 — Python on Your Odoo Server (Most Powerful)
1
Install the Gmail + GitHub libraries
pip install google-auth google-auth-oauthlib google-api-python-client pip install openpyxl pandas PyGithub
2
Python cron job — runs daily at 7am
from googleapiclient.discovery import build from google.oauth2.credentials import Credentials from github import Github import pandas as pd, base64, json def run_daily_update(): # 1. Connect to Gmail creds = Credentials.from_authorized_user_file('token.json') gmail = build('gmail', 'v1', credentials=creds) # 2. Find today's report result = gmail.users().messages().list( userId='me', q='subject:"Front Office Report" newer_than:1d has:attachment' ).execute() msg_id = result['messages'][0]['id'] msg = gmail.users().messages().get(userId='me', id=msg_id).execute() # 3. Download the Excel attachment for part in msg['payload']['parts']: if 'xlsx' in part['filename']: data = gmail.users().messages().attachments().get( userId='me', messageId=msg_id, id=part['body']['attachmentId'] ).execute() xlsx_data = base64.urlsafe_b64decode(data['data']) break # 4. Parse + rebuild dashboard # ... (same logic as Claude does — parse IDS, F.O.R., etc.) new_html = build_dashboard(xlsx_data) # 5. Push to GitHub g = Github('YOUR_GITHUB_TOKEN') repo = g.get_repo('username/saken-dashboard') file = repo.get_contents('index.html') repo.update_file('index.html', f'Auto-update {date.today()}', new_html, file.sha) print('Dashboard updated successfully') # Add to crontab: 0 7 * * * python3 /path/to/daily_update.py
💡 CFO Recommendation
Given your Odoo server + IT background: start with Option 1 (Google Apps Script) — it takes 30 minutes to set up, costs nothing, and requires no server maintenance.

Once the flow is stable, migrate to Option 3 (Python on your Odoo server) — this gives you full control, lets you integrate directly with your Odoo data (occupancy vs invoicing), and sets the foundation for an AI-driven property management module you could productize as part of your IT business.
